# Interviewing

Use this reference before drafting the spec.

## Goal

Interview the user in depth about the feature until the implementation shape is clear enough to write a practical spec.

Focus on non-obvious questions, not generic ones.

## What To Ask About

Probe for:
- technical implementation constraints;
- integration points;
- UI and UX implications if relevant;
- migration strategy;
- compatibility requirements;
- validation and testing expectations;
- tradeoffs the user already has in mind;
- explicit non-goals and boundaries.

## Interview Style

- Ask concise but substantive questions.
- Prefer a few high-signal questions at a time instead of a long questionnaire.
- Continue iterating until major ambiguities are resolved.
- Do not stop after obvious questions if meaningful edge cases or workflow decisions remain unclear.

## Completion Condition

Stop interviewing once the spec can be written without relying on large unstated assumptions about behavior, scope, or validation.
